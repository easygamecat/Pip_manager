# Pip Manager GUI

Графическая утилита (Tkinter) для управления установленными Python-пакетами через pip.
Позволяет просматривать пакеты с версиями, удалять их по одному или группами,
группировать похожие имена (например `PyQt4`/`pyqt5` → группа `pyqt`) и открывать
страницу пакета на PyPI в один клик.

## Возможности

- 📋 Список всех установленных пакетов с версиями (загружается в фоне)
- ✅ Мультивыбор через чекбоксы: удаление одного, нескольких или всех сразу
- 🏷️ Информация о среде: версия Python, `venv`/`системный`, путь к интерпретатору
- 🔎 Поиск по имени
- 📂 Группы (>1 пакета): `python`/`python3`/`python3.11` собираются в одну группу;
  клик по группе в правой панели отмечает все её пакеты
- 🌐 Двойной клик по пакету — открывает страницу на PyPI и показывает полное описание
- 📝 Колонка «Описание» подгружается с PyPI (с кэшированием)
- 🔄 Автоматическая повторная проверка `pip` после обновления, чтобы убрать красную метку устаревшей версии

## Установка и запуск

Требуется Python 3.6+ (Tkinter входит в стандартную поставку).

```bash
cd pip_manager
pip install -r requirements.txt
python -m src.main
```

## Сборка в .exe

Утилита собирается в один исполняемый файл через PyInstaller:

```bash
pip install pyinstaller
build.bat
```

Результат: `dist\pip_manager.exe`. Tkinter встраивается автоматически, Python не нужен.

## Структура проекта

```
pip_manager/
├── .gitignore
├── README.md
├── requirements.txt      # pyinstaller для сборки
├── build.bat            # сборка в exe
├── pyproject.toml       # метаданные пакета
├── src/
│   ├── main.py          # точка входа (добавляет корень проекта в sys.path)
│   ├── core/            # логика без привязки к GUI
│   │   ├── pip_wrapper.py  # список/удаление/обновление пакетов, инфо о среде, family_of
│   │   └── pypi.py        # загрузка описаний с PyPI + кэш
│   └── ui/
│       └── app.py       # PipManagerApp — весь Tkinter-интерфейс
└── tests/
    └── test_pip_wrapper.py
```

Принцип разделения: `core/` не зависит от GUI, `ui/` пользуется `core/`. Это упрощает
внесение изменений и тестирование логики отдельно от интерфейса.
